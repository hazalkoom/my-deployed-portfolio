Project Overview

StreamLang is a complete programming language implementation from scratch, designed to solve the problem of complex, nested function calls in data processing pipelines. Instead of writing deeply nested functions that are hard to read and maintain, StreamLang uses a pipe-forward syntax (using the â–· operator) that makes data flow through transformations in a clean, linear fashion.

This project represents a full-stack language implementation, including lexical analysis, parsing, type inference, and interpretation. Built with ANTLR4 for grammar definition and parsing, Python for the interpreter engine, and deployed as a containerized application on Hugging Face Spaces using Docker.

Why StreamLang?

Technical Implementation

StreamLang is built using industry-standard compiler construction techniques, making it a production-quality language implementation:

1. Lexical Analysis (Tokenization)

The lexer breaks source code into tokensâ€”the smallest units of meaning. Using ANTLR4's lexer grammar, StreamLang recognizes keywords, operators, identifiers, literals, and whitespace.

2. Parsing (Syntax Analysis)

ANTLR4 generates a parser from the grammar definition, which builds an Abstract Syntax Tree (AST) representing the program's structure. The grammar defines the language's syntax rules, including operator precedence and associativity.

3. Type Inference & Checking

The interpreter performs local type inference, deducing types from context without requiring explicit type annotations. Type checking ensures operations are valid before execution, catching errors early.

4. Interpretation

The interpreter walks the AST, executing operations in the correct order. The pipe-forward operator (â–·) passes the result of the left expression as the first argument to the right function, creating the pipeline effect.

5. Standard Library

StreamLang includes built-in functions for common operations: arithmetic (add, subtract, multiply, divide), I/O (print), and data manipulation functions.

Challenges & Solutions

Building a complete programming language from scratch presented unique challenges that required deep understanding of compiler theory, language design, and software engineering principles.

Grammar Design & Parsing

Challenge: Designing a grammar that correctly handles operator precedence, associativity, and the unique pipe-forward syntax.

Solution: Used ANTLR4's powerful grammar definition syntax to create a clear, unambiguous grammar. Defined operator precedence explicitly and used left-recursive rules for left-associative operators like the pipe-forward operator.

Type Inference System

Challenge: Implementing a type inference system that can deduce types from context without explicit annotations.

Solution: Built a type inference engine that performs bottom-up type checking on the AST. Types are inferred from literals, propagated through operations, and validated at function call sites. Error messages clearly indicate type mismatches.

Pipe-Forward Operator Implementation

Challenge: Implementing the pipe-forward operator (â–·) to correctly pass the left expression's result as the first argument to the right function.

Solution: In the AST evaluation phase, the pipe-forward operator is handled specially: it evaluates the left expression, then calls the right function with the result as the first argument, followed by any additional arguments provided.

Immutable Data Structures

Challenge: Ensuring all data structures are truly immutable and preventing accidental mutations.

Solution: Implemented value semantics for all data types. Operations return new values rather than modifying existing ones. This ensures thread-safety and makes the language's behavior predictable.

Docker Containerization

Challenge: Packaging the language interpreter, ANTLR4 runtime, and playground UI into a single, deployable Docker container.

Solution: Created a multi-stage Dockerfile that installs dependencies, builds the ANTLR4 grammar, and sets up the Gradio UI. The container is optimized for size and startup time, making it suitable for Hugging Face Spaces deployment.

Error Handling & Debugging

Challenge: Providing clear, helpful error messages that help developers understand what went wrong.

Solution: Implemented comprehensive error handling with detailed error messages including line numbers, column positions, and context. Syntax errors, type errors, and runtime errors are all caught and reported with actionable information.
